## 一、课题目标与主要研究内容
本课题面向国内旅行者，完成“GoingTour 智能旅游规划系统”前端设计与实现。核心目标如下：

1. 构建可维护的 Vue 3 前端工程架构，支持模块化开发与持续迭代。
2. 实现“目的地选择-偏好输入-AI推荐-行程生成-编辑导出”的完整业务闭环。
3. 完成社区、足迹、心愿清单、个人中心等配套功能。
4. 提升可用性与安全性，包括路由鉴权、输入校验、内容安全过滤、异常兜底等。
5. 形成可部署版本，支持 Docker + Nginx 生产部署。

## 二、中期已完成工作（结合当前代码库）
截至2026年03月04日，前端主体功能已完成，完成度评估约为 **75%~80%**。

### 1. 工程与架构
1. 已完成 Vue 3 + Vite 7 + Element Plus + Pinia + Vue Router 4 技术栈搭建。
2. 已形成 `api / services / composables / pages / components / utils` 分层结构。
3. 已配置开发代理（`/api -> http://localhost:8080`）与路径别名（`@`）。

### 2. 核心业务功能
1. 用户模块：登录、注册、个人主页、个人设置等页面已实现。
2. 行程模块：
   - 目的地选择（城市浏览、检索、心愿清单联动）已实现。
   - 创建行程 5 步流程（基础信息、个性化偏好、推荐选择、智能生成、行程预览）已实现。
   - 行程编辑页已实现，可保存更新 AI 行程内容。
   - 行程导出已实现 Word 与图片导出。
3. 社区模块：社区广场、帖子详情、发布帖子流程已实现。
4. 足迹模块：足迹地图、访问城市展示、心愿清单展示已实现。

### 3. 工程质量与可运维能力
1. 路由守卫已实现登录校验与创建行程前置条件校验。
2. 已实现表单验证与 XSS 过滤工具，提升输入与渲染安全性。
3. 已实现统一请求封装（含 token 注入与异常处理）。
4. 已提供 Nginx 与 Docker 配置，可用于容器化部署。

### 4. 量化进展（代码统计）

6. 构建验证：2026年03月04日执行 `npm run build` 成功，Vite 7.0.6 打包通过（约 21.81s）。
