## 🧪 测试步骤

### **第1步：启动服务**
```bash
# 后端服务
cd goingtour-backend
mvn spring-boot:run

# 前端服务
cd goingtour-frontend  
npm run dev
```

### **第2步：基础测试**
1. 访问：`http://localhost:5173`
2. 进入"创建行程"页面
3. 填写基本信息：
   ```
   🎯 目的地：北京
   📅 日期：任意3-5天
   👥 人数：2人
   💰 预算：3000-5000元
   ```

### **第3步：偏好设置**
```
🏷️ 旅行标签：历史文化、美食体验
🏨 住宿偏好：舒适型酒店
🚶 节奏：慢节奏
🍽️ 口味：不辣、清淡
```

### **第4步：AI生成测试**
1. 点击"生成AI行程计划"
2. **观察进度提示**：应该看到8个阶段的进度展示
3. **等待生成**：通常需要60-120秒
4. **查看结果**：应该自动跳转到新的AI展示页面

### **❌ 错误处理测试：**
可以通过以下方式测试错误处理：
1. **网络错误**：断网后点击生成
2. **超时错误**：等待超过2分钟
3. **服务错误**：关闭后端服务后测试

应该看到对应的错误提示：
- ⏰ AI服务响应超时，请稍后重试
- 🌐 网络连接失败，请检查网络后重试
- 💳 AI服务余额不足，请联系管理员
