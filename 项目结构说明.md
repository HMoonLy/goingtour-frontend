# GoingTour 前端项目结构说明

## 🏗️ 已完成的项目架构

### 1. 路由系统 (`src/router/router.js`)
✅ **完整的路由配置**
- 🔐 认证页面路由（登录、注册）
- 🏠 主应用路由（个人中心、行程管理）
- 🛡️ 路由守卫（登录验证、权限控制）
- 📱 响应式路由（web + 移动端适配）

### 2. 状态管理 (`src/store/`)
✅ **Pinia状态管理系统**
- 👤 `user.js` - 用户状态管理（登录、个人信息、偏好设置）
- 🗺️ `trip.js` - 行程状态管理（行程生成、管理、分享）
- 💾 数据持久化（localStorage自动保存/恢复）

### 3. API接口层 (`src/api/`)
✅ **完整的接口封装**
- 🔧 `request.js` - Axios配置、请求/响应拦截器
- 👤 `user.js` - 用户相关API（已对接后端）
- 🗺️ `trip.js` - 行程相关API（接口定义，待后端实现）

### 4. 布局系统 (`src/layouts/`)
✅ **两套布局模板**
- 🔐 `AuthLayout.vue` - 认证页面布局（登录、注册）
- 🏠 `DefaultLayout.vue` - 主应用布局（导航栏、侧边栏）
- 📱 响应式设计（支持移动端）

### 5. 项目配置
✅ **完整的开发环境**
- ⚡ Vite配置（热重载、代理）
- 🎨 Element Plus UI库
- 🗺️ 预留Leaflet地图集成
- 🔧 开发工具配置
